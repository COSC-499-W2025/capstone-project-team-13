# Data Flow Diagram Level 0

The Level 0 diagram presents the Digital Artifact Mining system as a single high-level process, showing how it interacts with all external entities. The primary actor is the User (or Guest), who provides consent, credentials, and project file uploads into the system, and receives back dashboards, portfolios, resumes, and project reports. The system communicates bidirectionally with an AI Service (LLM) — sending project data for analysis and receiving back generated descriptions, extracted skills, and importance scores. The GitHub API is queried to retrieve contributor statistics for code-based projects, enriching the project analysis with commit history and authorship data. The Local File System serves as both a source of project files uploaded by the user and a destination for exported documents such as PDF and DOCX resume outputs. Together, these four external entities define the complete boundary of the system and the data flows that cross it.
