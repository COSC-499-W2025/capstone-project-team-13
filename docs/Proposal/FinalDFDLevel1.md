The Level 1 diagram breaks the system into its full set of internal processes, tracing data from initial user interaction through to final output generation. The process begins with the user granting consent and authenticating via login or signup, with configuration and privacy settings established at this stage to define which folders and file types are excluded from analysis. Once authenticated, the user selects a folder or uploads files directly, with filters applied to scope the analysis. A Guest Upload path also exists for unauthenticated users who want a one-time analysis without creating an account.
The selected files flow into the Directory Crawler, which traverses the folder structure and retrieves its contents. This data passes through a File Type Classifier and Data Processing module, which determine the project type — whether code, text, or media — before routing it to the appropriate processing branch. Code projects undergo language identification, keyword extraction, and ATS scoring. Text projects are processed through document reading and summarization. Media projects are handled by the media analyzer. In parallel, the Contribution and Skills Extraction module queries the GitHub API to retrieve contributor data and Git statistics, enriching the project record with authorship and collaboration metrics.
All processed outputs converge into the Project Data and Analyses node, which writes to the central Data Stores (SQLite database). With AI consent granted, the Analyze Project Data with LLM process sends this data to the external AI Service, which returns generated descriptions, extracted skills, importance scores, and summaries. These results feed a right-side output column where dedicated processes generate project rankings, skills timelines and chronology, resume bullet points, and portfolio summaries — all stored back into the Data Stores for retrieval.
The lower section of the diagram handles user-facing features: the Portfolio and Web Showcase module presents ranked projects publicly or privately, the Evidence Manager allows users to attach metrics and testimonials to projects, the Interview Prep module generates STAR-format answers from project data, the Analytics and Skills Insights module provides timeline views and co-occurrence analysis, and the Deletion Manager handles safe removal of projects and AI cache data. Final outputs — including project reports and ranked portfolios — are returned to the user, completing the end-to-end data flow.
